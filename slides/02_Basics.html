Which should I use?

  • Attribute: special cases
  • Class: your goto selector!
  • Type: rarely

Box Model

padding, border, margin

box-sizing

What gets included in width and height calculation

  • content-box
  • border-box

content-box

  • only content of the element is included
  • padding, border and margin are added on top

border-box

  • content, padding and border are included
  • only margin is added on top
  • mostly the setting we want but not default
                    
                        * {
                            box-sizing: border-box;
                        }

Display

inline vs block

block

  • Uses the box model as described
  • Tries to expand to 100% width
  • Pushes other elements to the next line

inline

  • Height and width settings have no effect
  • Line-height has an effect
  • Paddings/Marigins/Borders will not push away elements above/below

Measuring in CSS

Possible Values

%, px, pt, em, rem, vh, vw

cm, mm, in, vmin, ex

%, px

  • Most used units
  • px for static spaces
  • % to adapt to whatever your parent element does
  • Special Case: % for height

em, rem

  • font-size dependent
  • em uses current element font size
  • rem uses root element font size
  • perfect for responsive design (overwritten font-sizes, accessibility settings)

vh, vw, vmin

  • viewport dependent sizes
  • vh = height, vw = width, vmin = whichever is smaller
  • 100 is full width/height
