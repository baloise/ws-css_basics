Which should I use?

  • Inline Styles: NEVER!
  • ID: rarely
  • Pseudo: special cases
  • Attribute: special cases
  • Class: your goto selector!
  • Type: rarely
  • Universal: Almost never

Box Model

padding, border, margin

box-sizing

What gets included in width and height calculation

  • content-box
  • border-box

content-box

  • only content of the element is included
  • padding, border and margin are added on top

border-box

  • content, padding and border are included
  • only margin is added on top
  • mostly the setting we want but not default
                    
                        * {
                            box-sizing: border-box;
                        }

Display

inline vs block

block

  • Uses the box model as described
  • Tries to expand to 100% width
  • Pushes other elements to the next line

inline

  • Height and width settings have no effect
  • Line-height has an effect
  • Paddings/Marigins/Borders will not push away elements above/below

Measuring in CSS

Possible Values

%, px, pt, em, rem, vh, vw

cm, mm, in, vmin, ex

%, px

  • Most used units
  • px for static spaces
  • % to adapt to whatever your parent element does
  • Special Case: % for height

em, rem

  • font-size dependent
  • em uses current element font size
  • rem uses root element font size
  • perfect for responsive design (overwritten font-sizes, accessibility settings)

vh, vw, vmin

  • viewport dependent sizes
  • vh = height, vw = width, vmin = whichever is smaller
  • 100 is full width/height

User Agent Styles

Base Styling of each Browser

  • Browser specific
  • Overwritten by anything you set in your css
  • Except if you omit <!DOCTYPE>
  • Fallback when no styling
  • Pitfalls: paddings and margins

Visible in Dev Tools

reset.css/normalize.css

  • Overwrite all common UA Stylesheets
  • For all browsers
  • Start from scratch with your application styles

reset vs normalize

  • reset removes all browser styling
  • normalize tries to make browser styling consistent
  • http://nicolasgallagher.com/about-normalize-css/

Best Practices

Classes instead of long selectors

                    
                        // DO
                        .my-button {}
                        .my-button:hover {}
                        .my-button__icon {}

                        // DON'T
                        body div section > button {}
                        body div section > button:hover {}
                        body div section > button i {}

Check out http://getbem.com/.

Inside out styling

  • Start from inside with paddings/borders/margins
  • Don't fix containers to height and try to center content
  • Set a line-height (usually somewhere around 1.3)

As few properties as possible

  • Set as few CSS properties as possible for your classes
  • Every additional property can have side effects for your children
  • Often you hunt problems introduced by parent styles
  • Don't blindly copy stuff from StackOverflow et al.
